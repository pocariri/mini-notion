# mini notion

개인 업무 관리를 위한 미니 노션 — PRD(`../02-prd.md`)와 Claude Design 와이어프레임(Mini Notion Wireframes)을
하나의 Next.js 서비스로 결합한 구현입니다.

## 실행

```bash
npm install
npm run dev
```

브라우저에서 [http://localhost:3000](http://localhost:3000) 을 엽니다.

Supabase 연결이 필요합니다. `.env.example`을 참고해 `.env.local`에 아래를 채우세요.

```
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=...
```

## 페이지 구성

| 경로 | 화면 | 와이어프레임 |
|---|---|---|
| `/login` | 로그인 — 중앙 카드, Google 계속하기 | 1d |
| `/workspace` | 업무 페이지 — 좌측 레일 + 글 목록 + 글 상세(3-pane), `/page` 슬래시 메뉴, 빈 상태 환영 화면 | 1a + 1b |
| (workspace 내 상세) | 페이지 상세 — 제목·내용 편집, 자동 저장, 삭제 | 1f · 1g |
| `/me` | 마이 페이지 — 별명 변경(#n/20), 프로필 이미지, 계정 탭(로그아웃·초기화) | 1i |

## 주요 기능

- **`/page` 슬래시 명령**: 목록 위 프롬프트에 `/`를 입력하면 슬래시 메뉴가 열리고, `page`로 새 글이 생성됩니다.
  일반 텍스트를 입력하고 Enter를 누르면 그 텍스트를 제목으로 새 글이 만들어집니다.
- **페이지 CRUD**: 생성 → 제목·내용 편집(입력이 멈춘 뒤 자동 저장) → 삭제(확인 후 영구 삭제).
  삭제는 되돌릴 수 없습니다.
- **검색**: 제목+내용 부분 일치.
- **마이 페이지**: 별명(최대 20자, `#n/20` 카운터), 프로필 이미지(5MB 이하), 로그아웃, 데이터 초기화

## 스타일

Mini Notion 디자인 시스템 토큰을 그대로 사용합니다 — Pretendard Variable(로컬 폰트),
웜 뉴트럴 그레이 + 단일 인디고 바이올렛 액센트(`#6a5df0`), 헤어라인 보더, 부드러운 라운드,
Lucide 라인 아이콘. 토큰은 `app/globals.css`의 `:root`에 정의되어 있습니다.

## 데이터와 인증

- **인증**: 실제 Google OAuth(Supabase Auth, PKCE). 로그인한 사용자만 페이지를 만들 수 있습니다.
- **저장**: 페이지는 Supabase `public.page` 테이블에, 프로필은 `public.profile`에 저장됩니다.
  같은 계정이면 기기·브라우저가 달라도 같은 페이지가 보이고, 브라우저 데이터를 지워도 남습니다.
  `localStorage`에는 Supabase 인증 세션만 남습니다.
- **접근 제어는 DB에서 강제됩니다.** `public.page`의 RLS 정책이 본인 소유 행만 조회·생성·수정·삭제하도록
  제한합니다. 클라이언트의 필터는 편의를 위한 것이지 보안 경계가 아닙니다.
- 모든 영속 상태는 `lib/store.tsx` 한 곳을 통해서만 읽고 씁니다. 컴포넌트는 Supabase를 직접 호출하지 않습니다.

## 현재 구현의 한계 (MVP · 운영 비용 0원)

- **즐겨찾기·휴지통·수정 시각이 없습니다.** 확정된 `public.page` 저장 구조(식별자·생성 시각·제목·내용·소유자)에
  이를 담을 칼럼이 없어 기능을 두지 않았습니다. 삭제는 확인 후 즉시 영구 삭제이며 복원 경로가 없습니다.
  목록 정렬과 시각 표시는 생성 시각만 사용합니다.
- **첫 로그인 시 예시 글을 만들지 않습니다.** 새 사용자는 빈 목록에서 시작합니다.
- **오프라인 작성과 동시 수정 충돌 해결은 범위 밖입니다.** 같은 페이지를 두 곳에서 고치면 나중에
  저장된 내용이 최종 상태가 됩니다.
- **페이지 공유·협업이 없습니다.** 모든 페이지는 비공개입니다.
