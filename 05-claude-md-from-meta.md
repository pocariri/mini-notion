# 프롬프트: CLAUDE.md에 "디자인 작업 전 DESIGN.md 필독" 지시사항 추가

> 이 파일은 **실행용 프롬프트**입니다. 아래 "프롬프트 본문"을 그대로 Claude Code에 입력하면,
> 프로젝트의 `CLAUDE.md`에 **모든 디자인 작업 시 `DESIGN.md`를 먼저 읽고 진행하라**는
> 지시사항이 추가됩니다.
>
> 워크플로우 상 위치: `01-prd.txt` → `02-prd.md`(PRD) → `03-*.webp`(레퍼런스 디자인) → 앱 구현 →
> `04`로 `DESIGN.md` 역추출 → **`05` 이 프롬프트로 `DESIGN.md`를 CLAUDE.md 규칙에 연결**.

## 프롬프트 본문 (여기부터 복사해서 사용)

이 프로젝트의 `CLAUDE.md`에 **디자인 관련 작업을 시작하기 전에 반드시 프로젝트 루트의
`DESIGN.md`를 먼저 읽고 진행하라**는 지시사항을 추가한다.

### 0. 절대 규칙 — 반드시 준수

1. **기존 내용 보존.** 현재 `CLAUDE.md`에 있는 내용(예: `@AGENTS.md` 임포트)은 삭제·수정하지 않고
   그대로 유지한다. 새 지시사항은 기존 내용 아래에 **추가**만 한다.
2. **`@DESIGN.md` 자동 임포트 금지.** `CLAUDE.md`에 `@DESIGN.md` 임포트 구문을 넣으면
   매 세션마다 대용량 문서 전체가 컨텍스트에 로드된다. 임포트가 아니라
   **"디자인 작업 시 `DESIGN.md`를 읽어라"는 조건부 지시문**으로 작성한다.
3. **사전 확인.** 작성 전에 `DESIGN.md`가 실제로 프로젝트 루트에 존재하는지, 그리고
   목차(어떤 섹션이 있는지)를 확인한다. 지시문에서 문서의 성격을 정확히 소개할 수 있어야 한다.
4. **간결하게.** `CLAUDE.md`는 매 세션 로드되는 파일이므로 추가 분량은 필요한 만큼만.
   장황한 설명 대신 명확한 규칙 몇 줄로 작성한다.

### 1. 추가할 지시사항에 반드시 포함할 내용

새 섹션(예: `## Design Rules`)을 만들고 아래를 담는다.

- **트리거 조건**: 다음 중 하나라도 해당하면 "디자인 작업"으로 간주한다 —
  UI 컴포넌트 생성/수정, 스타일(CSS/클래스) 변경, 색상·타이포·간격·반경·그림자 등
  디자인 토큰 사용, 레이아웃/화면 구조 변경, 새 화면 추가.
- **행동 규칙**: 디자인 작업이면 코드를 쓰기 전에 프로젝트 루트의 `DESIGN.md`를 먼저 읽는다.
  전체 정독이 부담이면 최소한 관련 섹션(토큰, 해당 컴포넌트, 해당 화면)은 반드시 읽는다.
- **준수 규칙**: 새로운 색상·수치를 임의로 만들지 말고 `DESIGN.md`에 정의된
  기존 토큰(`app/globals.css`의 CSS 변수)을 사용한다. 문서에 없는 값이 필요하면
  기존 체계와 일관되게 추가하되 그 사실을 사용자에게 알린다.
- **동기화 규칙**: 디자인에 실질적 변경(토큰 추가/변경, 컴포넌트 변형·상태 추가 등)이
  생기면 `DESIGN.md`도 함께 갱신하여 코드와 문서가 어긋나지 않게 한다.

### 2. 작성 형식

- `CLAUDE.md`는 영어·한국어 중 기존 파일의 언어 관례를 따르되, 규칙이 모호하지 않게 쓴다.
- 마크다운 섹션 제목 + 짧은 불릿 목록 형태로 작성한다.
- `DESIGN.md`, `app/globals.css` 같은 파일 경로는 인라인 코드로 표기한다.

### 3. 마지막 단계 — 자기 검증

작성 후 아래를 실제로 점검하고 결과를 보고한다.

- [ ] 기존 `CLAUDE.md` 내용(`@AGENTS.md` 등)이 그대로 보존되어 있다.
- [ ] `@DESIGN.md` 자동 임포트 구문이 **없다** (조건부 지시문으로만 참조).
- [ ] 트리거 조건 / 행동 규칙 / 준수 규칙 / 동기화 규칙이 모두 포함되어 있다.
- [ ] `DESIGN.md`가 실제로 존재하며, 지시문이 가리키는 경로가 정확하다.

### 4. 산출물

- 수정된 `CLAUDE.md` 한 개 파일.
- 작업 후, 추가된 지시사항 전문을 보여주고 어떤 판단(섹션 위치, 언어, 표현)을 했는지 짧게 보고한다.
