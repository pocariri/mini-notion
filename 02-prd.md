# PRD: 미니 노션 (Mini Notion) — 개인 업무 관리 웹 서비스

## 1. 요약 (Summary)

이 문서는 개인 업무 관리를 위한 웹 서비스 "미니 노션"의 제품 요구사항을 정의합니다. 사용자는 구글 계정으로 로그인한 뒤, 노션과 유사한 방식으로 업무 글(페이지)을 만들고, 수정하고, 삭제할 수 있습니다. 유료 구독 없이 나에게 꼭 맞는 개인용 업무 관리 툴을 직접 만들어 쓰는 것이 목표입니다.

## 2. 담당자 (Contacts)

| 이름 | 역할 | 비고 |
|---|---|---|
| (작성자) | 제품 오너 / 개발자 | 기획, 개발, 운영을 모두 담당하는 1인 프로젝트 |

> 개인 프로젝트이므로 모든 역할을 한 사람이 겸합니다. 협업자가 생기면 이 표를 갱신합니다.

## 3. 배경 (Background)

### 무엇에 관한 프로젝트인가?
노션(Notion) 같은 업무 관리 툴은 편리하지만, 개인이 쓰기에는 기능이 너무 많고 일부 기능은 유료입니다. 이 프로젝트는 개인 업무 관리에 꼭 필요한 최소 기능만 담은 "미니 노션"을 직접 만드는 것입니다.

### 왜 지금인가?
- 기존 툴의 유료화가 확대되면서, 무료로 쓸 수 있는 개인용 대안의 필요성이 커졌습니다.
- 직접 만들면 비용이 들지 않고, 내 업무 방식에 맞는 기능을 자유롭게 추가할 수 있습니다.
- 구글 OAuth 같은 인증 서비스와 무료 배포 환경이 잘 갖춰져 있어, 개인도 빠르게 서비스를 만들 수 있게 되었습니다.

## 4. 목표 (Objective)

### 목표
- 유료 구독 없이 사용할 수 있는 개인 전용 업무 관리 웹 서비스를 만든다.
- 개인 프로젝트이므로, 앞으로 나에게 필요한 기능을 자유롭게 확장할 수 있는 기반을 갖춘다.

### 왜 중요한가?
- **비용 절감**: 기존 유료 툴을 대체하여 구독 비용을 없앱니다.
- **맞춤화**: 남이 정한 기능이 아니라, 내 업무 흐름에 맞는 기능만 담습니다.
- **확장성**: 직접 만든 서비스이므로 원하는 기능을 언제든 추가할 수 있습니다.

### 핵심 결과 (Key Results)
| # | 핵심 결과 | 측정 방법 |
|---|---|---|
| KR1 | 구글 로그인으로 회원가입·로그인이 가능하다 | 로그인 성공 후 업무 페이지 진입 확인 |
| KR2 | 글 생성 → 작성 → 삭제의 핵심 흐름이 오류 없이 동작한다 | 핵심 시나리오 수동 테스트 통과 |
| KR3 | 기존 유료 툴 대신 이 서비스로 개인 업무를 관리한다 | 출시 후 4주간 주 3회 이상 실사용 |
| KR4 | 운영 비용 0원을 유지한다 | 무료 티어 범위 내 배포·운영 확인 |

## 5. 시장 세그먼트 (Market Segments)

### 누구를 위해 만드는가?
- **1차 사용자**: 개인 업무를 관리하고 싶은 사용자 (우선은 제작자 본인)
- **해결하려는 일(Job)**: "흩어져 있는 개인 업무를 한곳에 적어 두고, 쉽게 찾아보고 정리하고 싶다"

> 시장은 인구 통계가 아니라 사람들이 해결하려는 문제로 정의합니다. 이 서비스의 시장은 "가볍고 무료인 개인 업무 기록 공간이 필요한 사람"입니다.

### 제약 조건
- 개인 프로젝트이므로 개발 리소스는 1인, 운영 비용은 0원에 가깝게 유지해야 합니다.
- 팀 협업, 공유, 권한 관리 등 다인용 기능은 범위에서 제외합니다.

## 6. 가치 제안 (Value Propositions)

| 구분 | 내용 |
|---|---|
| 해결하는 일 | 개인 업무를 글 단위로 기록하고 관리 |
| 얻는 것 (Gains) | 무료 사용, 단순한 화면, 내 방식대로 확장 가능한 툴 |
| 피하는 고통 (Pains) | 구독 비용, 과도하게 복잡한 기능, 남의 서비스 정책 변화에 휘둘리는 것 |
| 기존 대안 대비 강점 | 노션·기타 유료 툴 대비: 완전 무료, 필요한 기능만 존재, 직접 수정·확장 가능 |

**가치 곡선 관점**: 기능의 개수와 협업 기능은 기존 툴보다 과감히 줄이고(제거·감소), 단순함·무료·개인 맞춤화는 기존 툴보다 크게 높입니다(증가·창조).

## 7. 솔루션 (Solution)

### 7.1 UX / 화면 흐름

```
[로그인 페이지] --구글 로그인--> [업무 페이지(글 목록)]
                                     │
      ┌──────────────────────────────┼─────────────────┐
      │                              │                 │
  "/page" 입력                    글 클릭           프로필 메뉴
      │                              │                 │
      ▼                              ▼                 ▼
  [새 글 생성]                  [글 상세 페이지]     [마이 페이지]
                                제목·내용 입력       별명 변경
                                글 삭제             프로필 이미지
```

### 7.2 주요 기능 (Key Features)

#### 1) 로그인 페이지
| 기능 | 설명 |
|---|---|
| 구글 로그인 | 구글 계정(OAuth)으로 로그인한다. 별도 회원가입 절차 없이 첫 로그인 시 계정이 생성된다. |

#### 2) 마이 페이지
| 기능 | 설명 |
|---|---|
| 별명 변경 | 서비스에서 표시될 별명(닉네임)을 자유롭게 변경할 수 있다. |
| 프로필 이미지 | 프로필 이미지를 등록·변경할 수 있다. |

#### 3) 업무 페이지
| 기능 | 설명 |
|---|---|
| 새 글 생성 | 입력창에 `/page`를 입력하면 새 글이 생성된다. (노션의 슬래시 명령어 방식) |
| 글 상세 이동 | 글 목록에서 글을 클릭하면 해당 글의 상세 화면으로 이동한다. |
| 글 작성·수정 | 글 상세에서 제목과 내용을 입력·수정할 수 있다. |
| 글 삭제 | 필요 없는 글을 삭제할 수 있다. |

### 7.3 기술 (Technology)

- **인증**: 구글 OAuth 2.0 기반 소셜 로그인
- **데이터**: 사용자(별명, 프로필 이미지), 글(제목, 내용, 작성자, 작성/수정 시각)을 저장하는 저장소 필요
- **배포**: 무료 티어로 운영 가능한 호스팅 환경 사용 (운영 비용 0원 목표)

> 구체적인 프레임워크·데이터베이스 선택은 구현 단계에서 결정합니다.

### 7.4 가정 (Assumptions) — 검증 필요

| # | 가정 | 검증 방법 |
|---|---|---|
| A1 | 글 생성·수정·삭제만으로도 개인 업무 관리가 충분하다 | 출시 후 본인이 실제로 계속 사용하는지 확인 |
| A2 | 구글 로그인만 제공해도 로그인 수단으로 충분하다 | 사용 중 다른 로그인 수단이 필요한지 확인 |
| A3 | `/page` 슬래시 명령 방식이 버튼보다 편리하다 | 실사용 후 불편하면 생성 버튼 추가 검토 |
| A4 | 무료 티어 인프라로 개인 사용량을 감당할 수 있다 | 운영하며 사용량·비용 모니터링 |

## 8. 릴리스 (Release)

### 1차 버전 (MVP) — 이 문서의 범위
- 구글 로그인
- 마이 페이지: 별명 변경, 프로필 이미지
- 업무 페이지: `/page`로 글 생성, 글 클릭 시 상세 이동, 제목·내용 입력, 글 삭제

### 이후 버전 (아이디어)
- 글 검색, 태그·폴더 분류
- 마감일·체크리스트 등 할 일(Task) 관리 기능
- 마크다운·블록 에디터 등 편집 기능 강화
- 개인에게 필요한 기능을 사용하면서 발견되는 대로 추가 (본 프로젝트의 핵심 취지)

### 예상 기간
- 정확한 날짜 대신 상대적 기간으로 계획합니다.
- **MVP**: 수 주 이내 (1인 개발 기준, 로그인 → 글 CRUD → 마이 페이지 순으로 구현)
- **이후 버전**: MVP 실사용 후 필요가 확인된 기능부터 점진적으로 추가
