# Quickstart: 랜덤 고양이 커버 이미지 — 검증 가이드

**Date**: 2026-07-13 | **Spec**: [spec.md](./spec.md) | **Plan**: [plan.md](./plan.md)

구현이 스펙을 만족하는지 끝까지(end-to-end) 확인하는 절차다. 자동 테스트(1)와 실제 구동 검증(2)을 모두 통과해야 완료로 선언한다(헌법 V·VI).

## 0. 사전 준비

```bash
npm install        # 최초 1회
```

## 1. 자동 테스트 (TDD 결과 확인)

```bash
npm run test:run
```

**기대 결과**: 기존 테스트(`lib/format`, `lib/store`, `Editor`) + 신규 `CatCover` 테스트 전부 통과, 경고 없음. 신규 테스트가 검증하는 동작은 research.md D6 목록(스켈레톤 초기 표시 / load→이미지 / error→폴백 / src 계약 / 리렌더 불변·리마운트 초기화 / Editor 통합) 그대로다.

## 2. 실제 구동 검증

```bash
npm run dev
```

브라우저에서 `http://localhost:3000` → 로그인 → 워크스페이스로 이동한 뒤, 아래 시나리오를 순서대로 확인한다.

### 시나리오 A — 기본 표시 (US1, SC-001)

1. 글 목록에서 글 하나를 연다.
2. **확인**: 제목 입력창 바로 위, 본문 폭과 같은 커버 영역에 고양이 사진이 표시된다.
3. **확인**: 이미지가 나타날 때 제목·본문이 아래로 밀리지 않는다(커버 자리가 처음부터 확보됨).
4. 다른 글로 전환한다. **확인**: 커버가 다시 로드되고(잠깐 스켈레톤) 새 고양이가 나온다.
5. 같은 글을 닫았다 다시 연다. **확인**: (거의 항상) 다른 고양이 — 매 진입 랜덤(FR-009).

### 시나리오 B — 스켈레톤 로딩 (US2, SC-002, SC-003)

1. DevTools → Network 탭 → Throttling을 `Slow 3G`로 설정.
2. 다른 글을 연다.
3. **확인**: 커버 자리에 스켈레톤(은은한 점멸)이 표시된다. 회전 스피너는 어디에도 없다.
4. **확인**: 스켈레톤이 떠 있는 동안 제목 입력창을 클릭해 즉시 타이핑할 수 있다.
5. **확인**: 이미지 도착 시 스켈레톤 자리에 그대로 교체되고 레이아웃이 덜컹이지 않는다.
6. **확인**: 타이핑(리렌더) 중 Network 탭에 cataas 재요청이 발생하지 않는다.

### 시나리오 C — 실패 폴백 (US3, SC-004)

1. DevTools → Network 탭 → `Offline` 설정.
2. 다른 글을 연다.
3. **확인**: 스켈레톤이 잠시 표시된 후 중립적인 빈 커버(고양이 라인 아이콘)로 전환된다. 오류 팝업·경고창 없음.
4. **확인**: 제목·내용 편집, 글 전환, 즐겨찾기·삭제가 평소대로 동작한다.
5. Throttling을 해제하고 다른 글을 연다. **확인**: 커버가 정상 로드로 복귀한다.

### 시나리오 D — 엣지 케이스

1. **빠른 전환**: 글 여러 개를 빠르게 연달아 클릭. **확인**: 마지막에 연 글의 커버만 표시되고, 이전 글 이미지가 뒤늦게 덮어쓰는 현상이 없다(FR-008).
2. **휴지통**: 글을 삭제 후 휴지통에서 연다. **확인**: 읽기 전용이어도 커버(로딩→이미지)가 동일하게 표시된다.
3. **빈 상태**: 선택을 해제(글 삭제 등)해 환영 화면을 띄운다. **확인**: 커버·스켈레톤이 표시되지 않는다.
4. **좁은 창**: 창 폭을 1024px 이하로 줄인다. **확인**: 커버가 본문 폭에 맞춰 줄고 가로 스크롤이 생기지 않는다.
5. **모션 감소**: OS의 "동작 줄이기(Reduce Motion)" 활성화 후 새 글 진입. **확인**: 스켈레톤이 점멸 없이 정적으로 표시된다.
