# Feature Specification: 내용 글자 수 카운터 (Content Character Counter)

**Feature Branch**: `001-content-char-count`

**Created**: 2026-07-10

**Status**: Draft

**Input**: User description: "내용 입력 칸에 내용 입력하면 글자 수 계산하는 기능을 만드려고 해. 우측 하단에 만들 예정이야."

## Clarifications

### Session 2026-07-10

- Q: 긴 글로 스크롤될 때 카운터 배치·표시 방식은? → A: 내용 편집 영역 우측 하단에 고정 표시 — 스크롤해도 항상 보임
- Q: 카운터 표시 형식은? → A: `128자` — 숫자 + 단위 접미사 "자"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 작성 중 글자 수 실시간 확인 (Priority: P1)

글을 작성하는 사용자가 내용 입력 칸에 텍스트를 입력·수정하는 동안, 화면 우측 하단에서 현재까지 입력한 글자 수를 실시간으로 확인한다.

**Why this priority**: 이 기능의 핵심이자 유일한 가치다. 입력에 맞춰 글자 수가 실시간으로 갱신되지 않으면 기능 자체가 의미를 잃는다. 이 스토리만 구현해도 완결된 MVP가 된다.

**Independent Test**: 워크스페이스에서 글 하나를 열고 내용 칸에 텍스트를 입력하면, 우측 하단 카운터의 숫자가 입력한 글자 수와 일치하도록 즉시 바뀌는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 내용이 비어 있는 글을 연 상태, **When** 사용자가 아무것도 입력하지 않음, **Then** 카운터는 "0자"를 표시한다.
2. **Given** 글 편집 화면, **When** 사용자가 내용 칸에 "안녕하세요"(5자)를 입력, **Then** 카운터는 "5자"를 표시한다.
3. **Given** 내용에 텍스트가 있는 상태, **When** 사용자가 일부 글자를 지움, **Then** 카운터가 즉시 줄어든 글자 수로 갱신된다.
4. **Given** 여러 줄과 공백이 포함된 내용, **When** 입력을 마침, **Then** 공백과 줄바꿈을 포함한 전체 글자 수가 표시된다.

---

### User Story 2 - 기존 글의 글자 수 확인 (Priority: P2)

이미 작성해 둔 글을 다시 열었을 때, 별도 입력 없이도 우측 하단에서 그 글의 현재 글자 수를 바로 확인한다.

**Why this priority**: 실시간 입력만큼 필수는 아니지만, 기존 글을 열자마자 분량을 파악하는 흔한 흐름을 지원한다. P1이 구현되면 자연히 따라오는 확장이다.

**Independent Test**: 내용이 저장된 기존 글을 선택해 열었을 때, 입력 동작 없이도 카운터가 그 글 내용의 글자 수를 표시하는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 내용이 저장된 글, **When** 사용자가 그 글을 열음, **Then** 카운터가 즉시 해당 내용의 글자 수를 표시한다.
2. **Given** 한 글을 보고 있는 상태, **When** 사용자가 다른 글로 전환, **Then** 카운터가 새로 연 글의 글자 수로 갱신된다.

---

### Edge Cases

- 내용이 비어 있을 때 → 카운터는 "0자"를 표시한다.
- 휴지통(읽기 전용) 상태의 글을 열었을 때 → 편집은 불가하더라도 조회는 가능하므로, 카운터는 해당 내용의 글자 수를 그대로 표시한다.
- 매우 긴 내용(예: 수만 자)일 때 → 전체 글자 수를 정확히 표시하고, 화면이 스크롤되어도 카운터는 편집 영역 우측 하단에 고정되어 계속 보이며, 숫자 자릿수가 늘어나도 배치·레이아웃이 깨지지 않는다.
- 공백·줄바꿈만 입력했을 때 → 공백과 줄바꿈도 글자로 계산되어 그 개수가 표시된다.
- 이모지·조합형 문자 등 → 문자열 길이(코드 유닛) 기준으로 계산한다(정밀한 grapheme 단위 계산은 범위 밖, Assumptions 참조).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 내용 입력 칸에 담긴 현재 텍스트의 글자 수를 내용 편집 영역 우측 하단에 **고정 표시**해야 하며, 내용이 길어 화면이 스크롤되더라도 카운터가 항상 보여야 한다(MUST).
- **FR-002**: 사용자가 내용 칸에서 글자를 입력·삭제할 때마다, 카운터는 별도 조작 없이 실시간으로 갱신되어야 한다(MUST).
- **FR-003**: 글자 수는 공백과 줄바꿈을 포함한 전체 글자 수로 계산해야 한다(MUST).
- **FR-004**: 내용이 비어 있을 때 카운터는 "0자"를 표시해야 한다(MUST).
- **FR-005**: 사용자가 다른 글로 전환하면, 카운터는 새로 표시되는 글의 글자 수로 갱신되어야 한다(MUST).
- **FR-006**: 카운터는 오직 내용 입력 칸의 텍스트만 계산하며, 제목 등 다른 입력 값은 계산에 포함하지 않아야 한다(MUST).
- **FR-007**: 카운터는 읽기 쉽게 표시되되, 내용 편집 영역을 가리거나 편집 흐름을 방해하지 않아야 한다(MUST).
- **FR-008**: 카운터는 글자 수를 `<숫자>자` 형식(숫자 + 단위 접미사 "자")으로 표시해야 한다(MUST). 예: `128자`.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 내용 칸에 입력한 즉시 카운터 값이 바뀌어, 입력과 표시 사이의 지연을 사용자가 인지하지 못한다.
- **SC-002**: 표시된 글자 수가 실제로 입력된 글자 수(공백·줄바꿈 포함)와 100% 일치한다.
- **SC-003**: 사용자는 버튼 클릭 등 추가 조작 0회로 글자 수를 확인할 수 있다.
- **SC-004**: 글을 열거나 다른 글로 전환한 직후 카운터가 항상 올바른 값을 표시한다(잘못된 값 노출 0건).

## Assumptions

- 글자 수는 공백과 줄바꿈을 포함한 전체 문자 수(문자열 길이)로 계산한다. 단어 수나 공백 제외 수가 아니며, 이모지 등 서로게이트 페어는 코드 유닛 기준으로 세어질 수 있다(정밀한 grapheme 단위 계산은 범위 밖).
- 계산 대상은 내용 입력 칸의 텍스트뿐이며, 제목 입력은 포함하지 않는다.
- 카운터 값은 별도로 저장되지 않고, 현재 표시 중인 글 내용에서 즉석 계산되는 파생 값이다(새로운 영속 데이터가 아니다).
- "우측 하단"은 내용 편집 영역을 기준으로 한 우측 하단이며, 스크롤과 무관하게 고정(sticky)되어 항상 보이도록 하고, 기존 제품의 시각적 스타일과 일관되게 표시한다.
- 휴지통(읽기 전용) 상태의 글도 조회 시 글자 수를 표시한다.
- 카운터는 `<숫자>자` 형식으로 표시한다(예: `128자`). 숫자만 표시하거나 별도 레이블을 붙이지 않는다.
