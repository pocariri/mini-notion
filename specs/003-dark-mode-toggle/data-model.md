# Data Model: 다크모드 (Dark Mode)

**Spec**: [spec.md](./spec.md) · **Date**: 2026-07-16

## 영속 데이터: 테마 환경설정 (Theme Preference)

기기·브라우저 단위 설정(FR-011). 서버·DB·Supabase 미사용 — `public.profile`에 컬럼을 추가하지 않는다.

### 저장 형식 — `localStorage`

| 항목 | 값 |
|---|---|
| 키 | `mini-notion:theme` (기존 `mini-notion:posts` 네이밍 관례를 따름) |
| 값 | `"light"` \| `"dark"` — 두 리터럴만 유효 |
| 부재 | 키 없음 = "선택된 적 없음" 상태. 운영체제 설정이 유효 테마를 결정(FR-013) |
| 손상 | 유효 리터럴이 아닌 값은 "선택된 적 없음"과 동일하게 취급하고 오류를 내지 않는다(FR-017) |

계정(`user_id`)과 무관하며, 로그아웃·"모든 데이터 초기화"(FR-018)에도 삭제되지 않는다.
`resetAll()`이 지우는 키 목록에 이 키를 포함하지 않는 것이 불변 조건이다.

### 도출 상태 (런타임, 비영속)

| 이름 | 타입 | 정의 |
|---|---|---|
| `savedTheme` | `'light' \| 'dark' \| null` | localStorage에서 읽은 유효 값. null = 선택된 적 없음 |
| `systemDark` | `boolean` | `prefers-color-scheme: dark` 미디어 쿼리 일치 여부. 판별 불가 환경은 `false`(라이트) |
| `theme` (유효 테마) | `'light' \| 'dark'` | `savedTheme ?? (systemDark ? 'dark' : 'light')` |

유효 테마는 항상 이 한 줄 규칙으로만 도출한다. 별도 "system" 값을 저장하거나 3-상태 토글을 만들지 않는다(FR-014).

### 상태 전이

```
                     ┌──────────────────────────────┐
                     │  선택된 적 없음 (키 부재/손상)     │
                     │  유효 테마 = OS 설정 따름         │
                     │  OS 변경 → 유효 테마 즉시 반영     │  ← FR-016
                     └──────────┬───────────────────┘
                                │ 토글 조작 (현재 유효 테마의 반대로 저장)
                                ▼
              ┌─────────────┐  토글  ┌─────────────┐
              │ saved=light │ ◄────► │ saved=dark  │
              └─────────────┘        └─────────────┘
                OS 변경 무시(FR-015)    OS 변경 무시(FR-015)
```

- 토글은 언제나 **현재 유효 테마의 반대**를 저장한다. 예: 선택 없음 + OS 다크(유효 다크) → 토글 → `light` 저장.
- "선택된 적 없음"으로 되돌아가는 전이는 없다(스펙 범위 밖 — 사용자가 저장소를 직접 지우는 경우만 해당).
- 저장소 쓰기 실패(프라이빗 모드 등) 시에도 런타임 상태 전이는 일어난다. 지속만 안 될 뿐이다(FR-017).

### DOM 반영 (렌더 계약)

| 대상 | 값 |
|---|---|
| `<html data-theme>` | 유효 테마와 항상 일치 (`"light"` \| `"dark"`) |
| 설정 시점 | 첫 페인트 전 인라인 스크립트(FR-012) 및 이후 토글 시 즉시 |

CSS는 `[data-theme='dark']`에서 시맨틱 토큰을 재정의한다. 컴포넌트 CSS는 토큰만 참조하므로 변경되지 않는다.

## 불변 조건 (Invariants)

1. `document.documentElement.dataset.theme`은 언제나 유효 테마와 같다 — 다른 값·부재 상태로 페인트되는 프레임이 없다(FR-012).
2. `mini-notion:theme` 키는 사용자가 토글을 조작했을 때만 쓰인다. 첫 방문 렌더가 OS 값을 저장해 버리는 일은 없다(그래야 FR-016의 "선택 전 OS 추적"이 유지된다).
3. 업무 화면과 마이 페이지의 토글은 같은 스토어 상태를 구독한다 — 화면 간 이동 시 토글 표시가 항상 일치한다(FR-001).
4. `resetAll()` 후에도 `mini-notion:theme`은 남는다(FR-018).
5. 테마 상태 변경은 posts·user 상태를 건드리지 않는다(FR-004).
